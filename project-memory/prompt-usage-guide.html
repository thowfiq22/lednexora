Lednexora Prompt Usage Guide

Use this guide whenever you continue the project in Codex, Antigravity, or another AI coding tool.

Main rule: Work one page/module at a time. Do not let AI start the full project from a memory file. Do not remember everything manually. Open the needed prompt file, paste it into the AI tool, and let the AI read the project-memory files.

3. When To Use Prompt 26

Use 26-end-of-day-update-prompt.txt after each completed page/module, or at the end of the work day.

Best habit: after one page is completed and tested, use prompt 26. This updates the changelog, so a new chat can continue tomorrow without confusion.

4. If Chat Closes Or You Continue Later

Use:

23-continue-from-last-session.txt

This tells the AI to read the changelog, find the last completed work, and continue from the correct next task.

5. If There Is A Bug Or Issue

Use:

24-bug-fix-debugging-prompt.txt

Paste the bug details or screenshot after the prompt. The AI should:

  1. reproduce or inspect the issue
  2. identify the affected file/page
  3. fix only the related area
  4. test again
  5. update the changelog

6. If You Forget The Full Order

Open:

27-full-project-build-order.txt

This is only a checklist/reference file, not a normal prompt you need to paste every time.

7. Token Saving Rule

Tell AI to read 00-start-here.md first, then only the task-specific files. Do not ask it to reread all project-memory files for every page.

8. Simple Daily Workflow

  1. Start new chat with 00-fresh-new-chat-start.txt.
  2. Use the next page/module prompt from the order table.
  3. Let AI plan/build only that page/module.
  4. Use 25-testing-qa-prompt.txt if you want QA.
  5. Use 24-bug-fix-debugging-prompt.txt for bugs.
  6. Use 26-end-of-day-update-prompt.txt after completion.
  7. Next day, use 23-continue-from-last-session.txt.

9. Do Not Change These

  • Accepted scope: 17 structured interfaces.
  • Frontend stack: Next.js + TypeScript static export for shared hosting/cPanel.
  • Backend stack: PHP/MySQL only for CMS/admin/forms/onboarding.
  • Do not build public website pages as PHP templates.
  • Design direction: Rimini-style enterprise quality, blue/white theme, controlled gradients.
  • Font direction: Source Sans 3 primary, free Rimini-like style.
  • Zoho: included as 3 reduced pages.
  • SAP MBC: not a fixed design page unless explicitly approved later.
  • Admin: limited content update/admin panel, not a full page builder.
